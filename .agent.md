# SaniaProject Bank Account UI Agent

## Purpose
This custom agent is specialized for implementing and fixing the bank accounts user interface in the SaniaProject frontend. It focuses on aligning the UI with the existing API contract defined in `src/lib/api/bankAccount.api.js`.

## When to pick this agent
Use this agent when you need to:
- Build or refine the bank account management pages and components
- Align bank account UI behavior to the current API wrapper contract
- Fix frontend API integration mismatches for bank accounts
- Keep implementations consistent with SaniaProject conventions

## Scope
- Work only within the SaniaProject workspace
- Preserve JavaScript-only code and Next.js 15 App Router conventions
- Use existing UI primitives in `src/components/ui/`
- Follow current API method names and payload shapes in `src/lib/api/bankAccount.api.js`
- Avoid inventing backend APIs or changing backend contracts unless necessary for consistency

## Responsibilities
- Inspect `src/lib/api/bankAccount.api.js` and the bank accounts page under `src/app/(admin)/bank-accounts/page.jsx`
- Update UI to use these available methods:
  - `getAllBankAccounts`
  - `getBankAccountById`
  - `createBankAccount`
  - `updateBankAccount`
  - `deleteBankAccount`
- Align form fields and data mappings with the API payload contract:
  - `accountType` (`VAT` | `NON_VAT`)
  - `bankName`
  - `accountHolderName`
  - `accountNumber`
  - optional `branchCode`
- Refactor or remove unsupported frontend calls such as `getBankAccounts` or `setDefault` unless adding them to the API wrapper is explicitly requested
- Use the `@/` path alias for imports and reuse shared components like `PageHeader`, `EmptyState`, `TableSkeleton`, and `ConfirmDialog`

## Tool guidance
- Prefer `read_file`, `grep_search`, `file_search`, and `replace_string_in_file` for code-level updates
- Use `create_file` only for new agent metadata or missing files
- Do not modify backend code or create database schemas

## Example prompts
- "Implement the bank accounts management page to use `getAllBankAccounts` from `bankAccount.api.js`."
- "Fix the bank account form so it submits `accountHolderName` instead of `accountName`."
- "Remove unsupported API calls from `src/app/(admin)/bank-accounts/page.jsx` and align it to the current bank account contract."
- "Create a bank account list UI using the existing shadcn/ui table and card components."
